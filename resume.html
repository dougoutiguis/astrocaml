Tutoriel de démarrage du devoir de programmation

1 Mise en place d'un dépôt git

1.1 En local

création du répertoire et initialisation de git 

mkdir tuto
cd tuto
git init

1.2 ou bien pour un dépôt créé depuis github (recommandé)

git clone <urldudepot>

2 Utilisation basique de git

2.1 Ajout d'un fichier

git add resume.org

2.2 Afficher le statut actuel du dépôt

Où l'on voit que le fichier a bien été ajouté pour faire partie du prochain commit. 

git status

2.3 Réaliser un commit pour sauvegarder une étape

Pour avoir un peu d'aide. 

git help commit

La commande commit permet de valider nos modifications et de leur associer un message explicatif. 

git commit -m "Ajout d'un compte-rendu"

Git nous informe alors du résultat de notre commande (un fichier modifié). 

[master (root-commit) 2bb1fab] Ajout d'un compte-rendu
1 files changed, 13 insertions(+), 0 deletions(-)
create mode 100644 resume.org

2.4 Vérifier les dernières modifications

git log

Commit de toutes les modifications sur les fichiers précédemment ajoutés. 

git commit -a -m "dernières modifs"

3 Commencer un projet caml

Création d'un fichier astro.ml 

(* Graphics.close_graph ();; *)
Graphics.open_graph " 350x350";;

print_string "          *** космонавт ***\n";;
print_string "       Salut jeune astronaute\n";;

À compiler comme toplevel comme ceci: 

ocamlmktop graphics.cma astro.ml -o FlappyCandyAstroCrush

Rajoutons un de nos fichiers d'images : 

git add images50.ml
git commit -m "ajout fichier images"

Et apprenons à afficher une image, dans un fichier dessiner.ml.

3.1 Le fichier dessiner.ml

Nous manipulons les images sous forme de matrices (des tableaux de tableaux) dont les éléments ont soit des couleurs soit des paires couleur, opacité.

On commence par une fonction pour récupérer les dimensions d'une matrice (hauteur et largeur). 

let dimensions m =
  let h = Array.length m in
  let l = Array.length m.(0) in
  (l, h);;

Le module Graphics permet de composer de nouvelles couleurs à partir de leurs composantes rouge verte et bleue (fonction Graphics.rgb). L'opération inverse n'est par contre pas fournie. Programmons la. 

let decompose c =
  let b = c mod 256 in
  let v = (c / 256) mod 256 in
  let r = (c / 65536) mod 256 in
  (r,v,b);;

On s'en sert pour mélanger deux couleurs selon un certain niveau d'opacité allant de 255 (opaque) à 0 (transparent). Il y a un bug dans le code suivant, saurez-vous le trouver ? (Il est corrigé dans le dépôt). 

let melanger c1 c2 op =
  let (r1,v1,b1) = decompose c1 in
  let (r2,v2,b2) = decompose c2 in
  let mix a b op =
    (op * a + (255 - op * b))/255 in
  let r3 = mix r1 r2 op in
  let v3 = mix v1 v2 op in
  let b3 = mix b1 b2 op in
  Graphics.rgb r3 v3 b3;;

La fonction matrixmap2 que nous avons construit en TP s'obtient très simplement comme ceci. 

let arraymap2 f ta tb =
  let g i b = f ta.(i) b in
  Array.mapi g tb;;

let matrixmap2 f =
  arraymap2 (arraymap2 f);;

Forts de la fonction melanger et de matrixmap2, il est facile d'écrire le code permettant de dessiner une image en tenant compte de la transparence. 

let dessiner_image img x y =
  let (largeur,hauteur) = dimensions img in
  let fond = Graphics.dump_image (Graphics.get_image x y largeur hauteur) in
  let melanger_tordu (c1,o) c2 = melanger c1 c2 o in
  Graphics.draw_image (Graphics.make_image (matrixmap2 melanger_tordu img fond)) x y;;

Ceci termine le fichier dessiner.ml. On l'ajoute au dépôt. 

git add dessiner.ml
git commit -a -m "ajout de la fonction de dessin d'images"

Le premier essai révèle un bug (notamment visible au fait que la transparence totale ne survient pas). On le corrige facilement. 

git commit dessiner.ml -m "correction d'un bug dans le calcul de la transparence"

3.2 Le fichier directions.ml

Comme les directions (les quatre points cardinaux) utilisées dans les déplacements sont nécessaires à la fois pour l'interprète et pour les graphismes, on les place dans un module à part. 

(* direction.ml *)
type direction = N | S | E | O;;

3.3 Un premier interprète pour nos tests (le fichier interprete.ml)

On rajoute un petit interpète maison (à augmenter à partir ce que vous avez appris à faire en TD/TP). Les instructions sont minimales : se déplacer dans les quatres directions, regrouper les instructions en blocs.

3.4 Retour au fichier principal (astro.ml)

Puis on commence à améliorer le jeu, voir directement dans les logs git, en particulier tout le fichier astro.ml.

3.5 Compilation complète

La compilation se fait alors comme ceci: 

ocamlmktop directions.cmo interprete.cmo graphics.cma images50.cmo dessiner.ml \
astro.ml -o FlappyCandyAstroCrush

Il est nécessaire de donner les modules dans l'ordre de leur utilisation (il faut respecter l'ordre de dépendance entre modules). On peut donner à cette ligne de commande les modules compilés, ou bien les fichiers sources l'un et l'autre fonctionne pour chaque fichier.

À l'ouverture de notre REPL (ou top-level) OCaml personnalisé, il reste malheureusement à ouvrir deux modules pour peupler l'espace de nom racine avec les types de l'interprète (et tout particulièrement la syntaxe du mini-langage) et pour donner accès à la fonction de jeu. Après seulement on peut commencer à jouer. 

open Interprete;;
open Astro;;
jouer (Bloc [Est; Est; Nord]);;
jouer (Bloc [Ouest; Nord;]);;
